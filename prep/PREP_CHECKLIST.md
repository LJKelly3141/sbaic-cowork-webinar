# Pre-Webinar Prep Checklist — Beyond the Chat Box

Work through this **the day of**, finishing 15 minutes before the scheduled start.

## 1. Render or refresh deliverables (day before)

- [ ] `cd slides && quarto render webinar.qmd` — confirm `webinar.html` is current
- [ ] `cd handout && quarto render handout.qmd` — confirm `handout.pdf` is current
- [ ] Push to GitHub repo so attendees can download

## 2. Cowork environment ready

- [ ] Claude Desktop **installed and signed in** on the presenting machine
- [ ] You're on a **Pro or higher plan** (Cowork requires it)
- [ ] Cowork mode opens cleanly (no setup wizard pending)
- [ ] **Fresh Cowork session** opened for the demo — no prior conversation
- [ ] At least one skill installed / available (so Demo 2's marketplace
      Option B has a fallback if save-as-skill is slow)

## 3. Sample folder ready

- [ ] `demo/sample-business-folder/` exists with 3–5 realistic small-business files:
  - [ ] A sample invoice (PDF or DOCX)
  - [ ] An email saved as `.txt` or `.eml`
  - [ ] A one-page memo (DOCX)
  - [ ] A small spreadsheet (XLSX)
  - [ ] *Optional:* A meeting agenda or note
- [ ] All files are **sanitized** (no real client info, no PII)
- [ ] Folder structure visible cleanly in Finder/Explorer (no clutter)

## 4. Files to have open

- [ ] `slides/webinar.html` — full-screen, presenter view
- [ ] **Claude Desktop in Cowork mode** — fresh session
- [ ] `demo/prompts.md` — for copy-paste
- [ ] Finder/Explorer window at `demo/sample-business-folder/` — visible
- [ ] `prep/RUN_OF_SHOW.md` — second monitor or printed

## 5. Dry-run the demo (10 min)

- [ ] Run Demo 1 prompt against the actual sample folder
- [ ] Confirm Cowork produces a 3-part report
- [ ] Run Demo 2 save-as-skill flow at least once — confirm skill registers
- [ ] **Start a fresh session** before going live (don't reuse the dry-run)

## 6. Audio + video + screen share

- [ ] Headset or quality mic plugged in and selected
- [ ] Mic level test
- [ ] Camera, lighting, framing checked
- [ ] Screen-share permissions confirmed
- [ ] Notifications **silenced** (Slack, email, calendar pop-ups)
- [ ] Do Not Disturb / Focus mode ON
- [ ] Browser tabs cleaned up — no embarrassing tab titles visible

## 7. Internet + power

- [ ] Wired ethernet preferred; if Wi-Fi, sit near router
- [ ] Laptop plugged in
- [ ] Phone hotspot ready as backup

## 8. Backup plan if Cowork fails live

- [ ] **Screen recording of a previous successful Demo 1 run** saved to
      `demo/reference-outputs/` — play it and narrate live if Cowork hangs
- [ ] **Screenshots** of the save-as-skill flow saved to
      `demo/reference-outputs/` — walk through them as static images
- [ ] Mental script: *"Cowork's tied up — let me show you what this looks
      like when it finishes"* + cut to recording / screenshots

## 9. Final 5 minutes before going live

- [ ] Bathroom break
- [ ] Water within arm's reach
- [ ] Slide deck on slide 1
- [ ] Cowork in a **fresh session**
- [ ] `prompts.md` open, scrolled to Demo 1
- [ ] Sample folder visible
- [ ] Deep breath. You've got this.

## 10. After the webinar

- [ ] Save Q&A transcript if available
- [ ] Note any Cowork quirks or surprises — update `demo/prompts.md`
- [ ] If a particularly good live output happened, save a screen recording
      or screenshots to `demo/reference-outputs/` for next time's backup
- [ ] If timing was off, update `prep/RUN_OF_SHOW.md`
- [ ] Note Q&A themes — could become a follow-up webinar topic
